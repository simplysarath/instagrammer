- [x] **6. Appwrite Functions: tag-product + remove-background**
  Spec ref: `spec.md > Appwrite Backend > Appwrite Functions`
  What to build: Create `appwrite_functions/tag-product/src/main.js` (Node.js 18): receives `file_id` from request body, downloads image from Appwrite Storage using Appwrite Node SDK, POSTs to Ximilar Fashion Tagging API (`https://api.ximilar.com/tagging/fashion/v2/tag`), maps response fields to ProductTags schema using the spec mapping (`category.name → garment_type`, `fabric.name → fabric`, `dominant_color.name → color`, `occasions[0].name → occasion`, `age_group.name → age_group`), returns `{ garment_type, fabric, color, occasion, age_group }`. On Ximilar failure, return empty tags object. Create `appwrite_functions/remove-background/src/main.js` (Node.js 18): receives `file_id`, downloads image, checks file size (if >10MB, compress to max 4MP using `sharp` before sending), POSTs to Remove.bg API, uploads result PNG back to `product-images` bucket, returns `{ new_file_id }`. On failure, returns `{ error: "bg_removal_failed" }`. Create `package.json` for each function. Deploy both functions to Appwrite (via Appwrite CLI or console). Set environment variables: `XIMILAR_API_KEY`, `REMOVEBG_API_KEY`, `APPWRITE_API_KEY`, `APPWRITE_ENDPOINT`, `APPWRITE_PROJECT_ID`. Implement `lib/features/upload/data/ximilar_service.dart` and `lib/features/upload/data/bg_removal_service.dart` in Flutter — each calls the corresponding Appwrite Function via Appwrite Functions SDK.
  Acceptance: Calling `tag-product` with a real `file_id` returns a tags object with at least `garment_type` populated. Calling `remove-background` with a real `file_id` returns `new_file_id` and the result image appears in Appwrite Storage. Both functions visible and active in Appwrite console.
  Verify: Use Appwrite console → Functions → Execute to manually invoke each function with a test `file_id`. Confirm expected response JSON.
